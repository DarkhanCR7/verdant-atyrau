# Verdant — сайт клиники лазерной косметологии

Полнофункциональный сайт для клиники лазерной косметологии **Verdant** (Атырау):
публичная часть (услуги, врачи, цены, контакты, онлайн-запись) и защищённая
админ-панель для сотрудников клиники (кто и когда записан, управление статусами
записей, врачами и услугами).

## Стек технологий

| Слой | Технология |
|---|---|
| Фреймворк | Next.js 16 (App Router), React 19, TypeScript (strict) |
| Стили | Tailwind CSS v4 |
| База данных | PostgreSQL |
| ORM | Drizzle ORM (+ drizzle-kit для миграций) |
| Аутентификация | Auth.js / NextAuth v5 (credentials, JWT-сессии, httpOnly cookies) |
| Валидация | Zod (общие схемы для клиента и сервера) |
| Формы | React Hook Form |
| Тесты | Vitest (unit) |
| Контейнеризация | Docker + docker-compose |

## Почему Drizzle, а не Prisma

Изначально предполагался Prisma, но в среде сборки нет доступа к
`binaries.prisma.sh` (закрытая сеть), откуда Prisma скачивает бинарные движки.
Drizzle — чистый TypeScript/JS ORM без бинарных зависимостей, полностью
эквивалентен по возможностям (типобезопасность, миграции, констрейнты,
индексы) и не имеет этой проблемы ни в разработке, ни в проде.

## Подключение к базе данных: обычный порт 5432 или HTTP (443)

Многие сети (корпоративный Wi-Fi, некоторые провайдеры/VPN) блокируют
исходящие подключения на порт 5432 — из-за этого сайт может зависать при
попытке достучаться до базы, даже если сама база в полном порядке. Чтобы это
не мешало разработке и не ломало сайт для конечных пользователей,
`src/db/index.ts` сам выбирает способ подключения:

- Если `DATABASE_URL` указывает на **Neon** (в строке есть `neon.tech`) —
  используется HTTP-драйвер Neon (`@neondatabase/serverless` +
  `drizzle-orm/neon-http`), который работает через порт 443 — тот же, что у
  обычных сайтов, его никто не блокирует.
- Для локального/Docker Postgres (обычный `DATABASE_URL` без `neon.tech`) —
  используется стандартный `postgres.js`-драйвер через порт 5432, как обычно.

Ничего вручную переключать не нужно — просто работает в зависимости от того,
что указано в `DATABASE_URL`.

**Важная оговорка:** это касается только самого работающего сайта (`npm run
dev`, `npm run build && npm run start`, и продакшена на Vercel). Команды
`npm run db:migrate` и `npm run db:seed` — это отдельная утилита
(`drizzle-kit`), она всегда подключается напрямую через порт 5432, в обход
этого фикса. Если твоя сеть блокирует 5432 — эти две команды всё ещё будут
зависать, даже когда сам сайт уже работает нормально. В этом случае
накатывай миграции и сид через веб-интерфейс **SQL Editor** в панели Neon
(там можно выполнить SQL прямо в браузере, без прямого сетевого
подключения с твоего компьютера) — то же самое сделали при первой настройке
базы для этого проекта.

## Реализованный функционал

**Публичная часть**
- Адаптивный дизайн для мобильных и десктопа
- Главная, услуги, врачи, цены, контакты (с картой)
- Онлайн-запись на приём: выбор услуги, врача, даты, времени; проверка занятых
  слотов в реальном времени; защита от двойного бронирования на уровне БД
  (уникальный индекс `doctor_id + date + time`)
- Форма обратной связи
- SEO: метатеги, `sitemap.xml`, `robots.txt`, семантическая разметка
- Доступность: видимый focus-ring, alt/aria-атрибуты, контрастная палитра

**Админ-панель (`/admin`)**
- Вход по email/паролю (bcrypt-хэш, rate limit на попытки входа)
- Роли `ADMIN` / `STAFF` (RBAC): STAFF видит записи, ADMIN дополнительно
  управляет врачами и услугами
- Дашборд со статистикой (записи сегодня / на этой неделе / ожидающие)
- Таблица записей с фильтрами (дата, статус, врач, поиск по имени/телефону),
  пагинацией и сменой статуса (`PENDING → CONFIRMED → COMPLETED / NO_SHOW / CANCELLED`)
- CRUD врачей и услуг (soft-delete — скрытие, а не физическое удаление)
- Audit log: кто и когда изменил статус записи / врача / услугу

**Безопасность**
- OWASP-заголовки (CSP, X-Frame-Options, X-Content-Type-Options, HSTS в проде,
  Referrer-Policy, Permissions-Policy) — `src/middleware.ts`
- Хэширование паролей bcrypt (12 раундов)
- httpOnly + SameSite=Lax сессионные cookie, `__Host-` префикс в проде
- Rate limiting на форму записи, форму обратной связи и попытки входа
- Валидация Zod на клиенте и сервере (все API-роуты валидируют вход заново)
- Honeypot-поле против ботов (тихий "успех" вместо явной ошибки)
- Защита от IDOR: доступ к admin API только по сессии, ID проверяются как UUID
- Privacy: IP-адрес никогда не хранится в открытом виде — только SHA-256 хэш
- SQL-инъекции невозможны в рамках ORM (параметризованные запросы Drizzle)

## Быстрый старт (локально)

```bash
cp .env.example .env
# отредактируйте .env — задайте DATABASE_URL, AUTH_SECRET (openssl rand -base64 32)

npm install
npm run db:migrate     # применить миграции
npm run db:seed        # заполнить демо-данными + создать админ-аккаунт
npm run dev
```

Приложение: http://localhost:3000
Админка: http://localhost:3000/admin/login

Данные для входа после сида (**обязательно смените пароль**):
- email: `admin@verdant-atyrau.kz`
- пароль: `ChangeMe123!`

Подробности — в `INSTALL.md` и `DEPLOYMENT.md`.

## Структура проекта (clean architecture)

```
src/
  app/            # страницы и API-роуты (Next.js App Router)
    admin/        # защищённая админ-панель
    api/          # REST API
  components/
    site/         # публичные UI-компоненты
    admin/        # компоненты админки
  server/         # слой репозиториев/сервисов — вся работа с БД
  lib/            # валидация, rate-limit, auth-хелперы, конфигурация
  db/             # Drizzle-схема, клиент, seed
  middleware.ts   # auth guard + security headers
tests/unit/       # Vitest unit-тесты
drizzle/          # SQL-миграции
```

## ⚠️ Перед запуском — проверь и замени

Часть данных заполнена по открытым источникам (2ГИС) или как разумное
предположение — обязательно проверь перед сдачей клиенту:

- **Часы работы** (`src/lib/clinic.ts`): у меня есть только "сегодня 10:00–18:00"
  из 2ГИС-карточки, полное расписание на неделю я предположил как
  Пн–Сб 10:00–18:00, Вс — выходной. **Уточни у клиента и поправь при
  необходимости.**
- **Врачи** — в базе сейчас 2 демо-специалиста-заглушки ("Врач-косметолог
  клиники", "Врач-хирург клиники"). Замени на реальных сотрудников через
  `/admin/doctors` (имя, специализация, стаж, фото).
- **Услуги** — 3 позиции (консультация хирурга, удаление бородавки/родинки)
  взяты из реальной карточки 2ГИС, остальные (биоревитализация, чистка лица,
  эпиляция и т.д.) — типичные для лазерной косметологии, но не подтверждены
  напрямую у клиента. Сверь полный список и цены через `/admin/services`.
- **Email** (`info@verdant-atyrau.kz`) — placeholder, замени на реальный
  рабочий адрес клиники.

## Что не реализовано (сознательно, по итогам уточнения задачи)

Изначальный запрос включал полноценный SaaS-масштаб (личные кабинеты
пациентов, email-верификация, password reset, и т.д.) — после уточнения было
решено сделать сайт-визитку с онлайн-записью и админкой для персонала, без
личных кабинетов пациентов. Также не подключены: тёмная тема, E2E-тесты
(Playwright), реальная отправка email/SMS-уведомлений (сейчас — заглушка с
`console.info`, легко подключить Resend/SES или SMS-провайдера).
